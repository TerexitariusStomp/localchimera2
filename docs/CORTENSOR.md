# Cortensor Network Integration

## Overview

Cortensor is a decentralized AI network for proof-of-useful-work (PoUW). The Chimera node integrates with Cortensor by wrapping the native `cortensord` binary and managing its lifecycle.

## Where Your Node Appears in the Frontend

Once fully set up, your node appears at:

**https://cortensor.network**

Navigate to:
- **"Network"** or **"Nodes"** tab — lists all registered and online miner nodes
- Your node ID (derived from the key generated by `cortensord`) will be shown with status:
  - `registered` — completed `tool register`
  - `verified` — completed `tool verify`
  - `online` — `minerv2` process is running and heartbeating

## Prerequisites

1. **Linux, macOS (ARM64), or Windows (Cygwin)**
2. **EVM wallet address** (Arbitrum Sepolia testnet) — must be **whitelisted by Cortensor Admin** before registration works
3. **Docker** (optional, for LLM engine container)

## Setup Steps

### Step 1 — Install Cortensor

```bash
curl -L https://github.com/cortensor/installer/archive/main.tar.gz -o cortensor-installer.tar.gz
tar xzfv cortensor-installer.tar.gz
cd installer

# Linux (Ubuntu 22.04 / Debian 12)
sudo ./install-linux.sh

# macOS (ARM64)
./install-osx-arm.sh
```

This creates:
- `~/.cortensor/bin/cortensord` — the daemon binary
- `~/.cortensor/.env` — environment / config file
- `~/.cortensor/llm-files/` — LLM model storage

### Step 2 — Get Whitelisted

**Critical:** Your EVM address must be whitelisted by Cortensor Admin **before** registration.

- Discord: https://discord.gg/cortensor
- Or contact Cortensor support on Telegram

Provide them with the `walletAddress` you will use.

### Step 3 — Configure Chimera

Edit `qvac/config.json`:

```json
"cortensor": {
  "enabled": true,
  "config": {
    "network": "arbitrum-testnet",
    "walletAddress": "0xYourWhitelistedEVMAddress",
    "cortensorHome": "~/.cortensor",
    "envFile": "~/.cortensor/.env",
    "cortensordPath": ""
  }
}
```

Leave `cortensordPath` empty to auto-detect.

### Step 4 — Generate Keys

```bash
# Or let the miner auto-generate on first start
cortensord ~/.cortensor/.env tool gen_key
```

### Step 5 — Register

```bash
# Via CLI (recommended first time)
cortensord ~/.cortensor/.env tool register

# Or via the Chimera node's upcoming API endpoint:
# POST /api/miners/cortensor/register
```

### Step 6 — Verify

```bash
cortensord ~/.cortensor/.env tool verify
```

### Step 7 — Start Mining

Start the Chimera node normally:

```bash
cd qvac && node src/index.js
```

The `CortensorMiner` will spawn `cortensord minerv2` as a child process.

## Miner Status API

```bash
curl http://localhost:3002/api/status
```

Look for:
```json
{
  "mining": {
    "minerStatus": {
      "cortensor": {
        "running": true,
        "walletConfigured": true,
        "walletAddress": "0x1234...abcd",
        "cortensordFound": true,
        "envFileFound": true,
        "registered": true,
        "verified": true
      }
    }
  }
}
```

## Troubleshooting

| Issue | Fix |
|---|---|
| `cortensord not found` | Run the Cortensor installer first |
| `Registration failed` | Address not whitelisted — contact Cortensor Admin |
| `Verification failed` | Run `register` first, ensure node is online |
| `Node not in frontend` | Must be registered + verified + `minerv2` running |

## Architecture

```
[Chimera NodeManager]
       └── CortensorMiner
              ├── Spawns: cortensord ~/.cortensor/.env minerv2
              ├── Stdout/Stderr piped to Chimera logs
              └── SIGTERM on stop (graceful shutdown)
```
